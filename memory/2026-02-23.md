## Solana Jupiter Bot - Incidente y Resolución

### Problema Detectado (9:00 AM Heartbeat)
- Bot detenido desde 3:36 AM (hace 5.5 horas)
- Último log: Ciclo 1, 9 estrategias, 8 posiciones paper (+$3.83)
- Proceso no encontrado

### Investigación
- No había errores en el log (el bot se detuvo limpiamente)
- Watchdog no estaba corriendo
- El bot tiene auto-reinicio interno (while True), pero sin watchdog no se reinicia si el sistema mata el proceso

### Causa Raíz
El **watchdog no estaba corriendo**. El watchdog es necesario para mantener el bot vivo si el sistema mata el proceso por:
- OOM (Out Of Memory)
- Systemd terminating the process
- Crash no capturado

### Solución Implementada
1. **Watchdog iniciado manualmente** (PID 216600 a las 9:48 AM)
   - Script: `~/.openclaw/workspace/Solana-Cripto-Trader/run_watchdog.sh`
   - Mantendrá el bot corriendo y lo reiniciará si falla

2. **Auto-start en .bashrc configurado**
   - Script: `~/.openclaw/workspace/Solana-Cripto-Trader/jupiter_bot_service.sh`
   - Se inicia automáticamente al abrir sesión de bash

3. **Servicio systemd preparado** (no instalado, requiere sudo)
   - Archivo: `/tmp/solana-jupiter-bot.service`
   - Habilitaría auto-start al boot del sistema
   - Requiere: `sudo cp /tmp/solana-jupiter-bot.service /etc/systemd/system/ && sudo systemctl enable solana-jupiter-bot`
